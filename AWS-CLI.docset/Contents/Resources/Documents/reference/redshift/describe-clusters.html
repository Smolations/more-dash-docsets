[ aws . redshift ]

describe-clusters¶

Description¶

Returns properties of provisioned clusters including general cluster properties, cluster database properties, maintenance and backup properties, and security and access properties. This operation supports pagination. For more information about managing clusters, go to Amazon Redshift Clusters in the Amazon Redshift Management Guide .

Synopsis¶

  describe-clusters
[--cluster-identifier <value>]
[--max-records <value>]
[--marker <value>]

Options¶

--cluster-identifier (string)

The unique identifier of a cluster whose properties you are requesting. This parameter isn't case sensitive.

The default is that all clusters defined for an account are returned.

--max-records (integer)

The maximum number of records that the response can include. If more records exist than the specified --max-records value, a marker is included in the response that can be used in a new describe-clusters request to continue listing results.

Default: 100

Constraints: Value must be at least 20 and no more than 100.

--marker (string)

An optional marker returned by a previous describe-clusters request to indicate the first cluster that the current describe-clusters request will return.

You can specify either a --marker parameter or a --cluster-identifier parameter in a describe-clusters request, but not both.
