authorize-cache-security-group-ingress¶

Description¶

Authorizes ingress to a CacheSecurityGroup using EC2 Security Groups as authorization (therefore the application using the cache must be running on EC2 clusters). This API requires the following parameters: --ec2-security-group-name and EC2SecurityGroupOwnerId.

Note

You cannot authorize ingress from an EC2 security group in one Region to an Amazon Cache Cluster in another.

Synopsis¶

  authorize-cache-security-group-ingress
--cache-security-group-name <value>
--ec2-security-group-name <value>
--ec2-security-group-owner-id <value>

Options¶

--cache-security-group-name (string)

The name of the Cache Security Group to authorize.

--ec2-security-group-name (string)

Name of the EC2 Security Group to include in the authorization.

--ec2-security-group-owner-id (string)

AWS Account Number of the owner of the security group specified in the --ec2-security-group-name parameter. The AWS Access Key ID is not an acceptable value.
